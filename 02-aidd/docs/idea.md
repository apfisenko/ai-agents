Моя идея — создать LLM-ассистента в виде Telegram-бота **в роли консультанта по путешествиям**.

Бот ведёт диалог с пользователем и помогает узнать про **интересные места**, **народные традиции**, **страны**, **достопримечательности** и смежные темы (культура, ориентиры для путешественника). Поведение и акценты задаются **системным промптом** LLM; техническая реализация — как в [vision.md](vision.md): тот же поток «сообщение → история → LLM → ответ», без отдельных баз знаний и внешних источников в MVP.

Технические детали реализации в этом файле не рассматриваются, зафиксирована только концепция сервиса.
