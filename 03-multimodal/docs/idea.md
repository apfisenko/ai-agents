# Идея продукта

**Персональный финансовый советник** в Telegram: бот ведёт **учёт доходов и расходов** по сообщениям из чата, помогает смыслом ответов (краткие комментарии, баланс по запросу), без отдельной бухгалтерской БД и без выгрузок в сторонние сервисы учёта в MVP.

Пользователь описывает траты и заработок **текстом**, **голосовыми сообщениями** или присылает **фото чеков**. Голос обрабатывается **без отдельного сервиса распознавания речи**: аудио передаётся в **мультимодальную LLM** (speech-to-text на стороне модели), после чего сценарий тот же, что для текста — **структурированные транзакции** и ответ пользователю. Бот извлекает данные из этих входов и накапливает их **в памяти процесса** (как и историю диалога): после перезапуска учёт обнуляется.

**Запись одной транзакции (логическая таблица):** дата и время, тип движения (доход / расход), сумма, **тип операции** (повседневные, периодические, разовые), **категория** (продукты, рестораны, такси, образование, путешествия и др. — модель может предложить новую, если не подходит готовый ярлык), **описание** (товар, услуга, контрагент, источник — всё, что нужно, чтобы позже понять суть).

**Техническая идея** (детали — [vision.md](vision.md)): тот же стек (Python, uv, aiogram, OpenAI SDK); текст, **аудио из голосовых** и картинки обрабатываются через совместимый с API вызов к LLM/VLM с **structured output** там, где это сценарий учёта; чеки — через **VLM** (модель с зрением); голос — через **мультимодальную модель с поддержкой аудио** в запросе (без отдельного ASR-сервиса). При желании **не отправлять финансы внешним провайдерам** — переключение на **локальный Ollama** (или иной OpenAI-совместимый endpoint) через конфигурацию, при условии что выбранная модель и endpoint поддерживают нужный режим (текст / vision / **audio**).

В этом файле не накладываются детали реализации модулей — только продуктовая концепция.
