# Идея продукта

**Персональный финансовый советник** в Telegram: бот ведёт **учёт доходов и расходов** по сообщениям из чата, помогает смыслом ответов (краткие комментарии, баланс по запросу), без отдельной бухгалтерской БД и без выгрузок в сторонние сервисы учёта в MVP.

Пользователь описывает траты и заработок **текстом** или присылает **фото чеков**. Бот извлекает из этого **структурированные транзакции** и накапливает их **в памяти процесса** (как и историю диалога): после перезапуска учёт обнуляется.

**Запись одной транзакции (логическая таблица):** дата и время, тип движения (доход / расход), сумма, **тип операции** (повседневные, периодические, разовые), **категория** (продукты, рестораны, такси, образование, путешествия и др. — модель может предложить новую, если не подходит готовый ярлык), **описание** (товар, услуга, контрагент, источник — всё, что нужно, чтобы позже понять суть).

**Техническая идея** (детали — [vision.md](vision.md)): тот же стек (Python, uv, aiogram, OpenAI SDK); текст и картинки обрабатываются через LLM с **structured output**; чеки — через **VLM** (модель с зрением); при желании **не отправлять финансы внешним провайдерам** — переключение на **локальный Ollama** (или иной OpenAI-совместимый endpoint) через конфигурацию.

В этом файле не накладываются детали реализации модулей — только продуктовая концепция.
